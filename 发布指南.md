# 个人网站发布到GitHub Pages指南

## 📋 前期准备

### 1. 安装Git

1. 访问 https://git-scm.com/download/win
2. 下载Windows版本的Git安装包
3. 运行安装程序，一路点击"Next"使用默认设置即可
4. 安装完成后，**重新打开**PowerShell或命令提示符

### 2. 配置Git（只需做一次）

打开PowerShell，运行以下命令（替换成你的信息）：

```powershell
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

## 🚀 发布步骤

### 步骤1：初始化Git仓库

在 `e:\个人网站` 文件夹中，右键选择"在终端中打开"或打开PowerShell，运行：

```powershell
cd e:\个人网站
git init
```

### 步骤2：添加文件到暂存区

```powershell
git add .
```

### 步骤3：提交更改

```powershell
git commit -m "初始提交：个人作品集网站"
```

### 步骤4：连接到GitHub仓库

```powershell
git remote add origin https://github.com/KOMA1106/KOMADES.github.io.git
```

### 步骤5：推送到GitHub

```powershell
git branch -M main
git push -u origin main
```

**注意**：第一次推送时，GitHub会要求你登录认证，按照提示操作即可。

## 🌐 启用GitHub Pages

### 1. 打开仓库设置
1. 访问你的GitHub仓库：https://github.com/KOMA1106/KOMADES.github.io
2. 点击顶部的 **Settings**（设置）标签

### 2. 配置Pages
1. 在左侧菜单中找到 **Pages**（在"Code and automation"部分）
2. 在 **Build and deployment** 区域：
   - **Source** 选择：`Deploy from a branch`
   - **Branch** 选择：`main` 分支，文件夹选择 `/ (root)`
3. 点击 **Save** 保存

### 3. 等待部署完成
- 通常需要1-5分钟
- 刷新页面，当看到绿色的 "Your site is live" 提示时就完成了！

## 🎉 访问你的网站

部署成功后，你的网站地址是：
**https://KOMA1106.github.io/**

## 📝 以后更新网站

当你修改了网站内容后，只需要运行：

```powershell
cd e:\个人网站
git add .
git commit -m "更新描述"
git push
```

GitHub Pages会自动重新部署！

## ❓ 常见问题

**Q: 推送时提示认证失败怎么办？**
A: GitHub现在需要使用Personal Access Token，可以在Settings -> Developer settings -> Personal access tokens中创建。

**Q: 网站显示404怎么办？**
A: 确保你的主文件名是 `index.html` 或 `portfolio.html`，并且在仓库根目录下。

**Q: 原图片文件夹要不要上传？**
A: 不需要，我已经在 `.gitignore` 中排除了 `portfolio-images-original/` 备份文件夹，只上传压缩后的图片。
